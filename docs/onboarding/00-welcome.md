# 0장. 시작하기 전에 — 이 프로젝트의 큰 그림과 공부 순서

> **이 교과서는 누구를 위한 것인가요?**
> 이 프로젝트(fGPU)에 새로 합류한 개발자를 위한 인수인계용 교과서입니다.
> 다음 수준을 가정하고 썼습니다:
> - C 언어는 거의 처음이에요. `malloc`이 뭔지 몰라도 괜찮습니다.
> - Python은 조금 읽을 줄 알아요.
> - Java **Spring은 다뤄봤어요** (그래서 백엔드는 Spring에 빗대어 설명합니다).
> - 터미널·쉘 스크립트(`.sh`)·Docker는 잘 몰라요.
>
> 걱정 마세요. **필요한 사전 지식을 전부 이 교과서 안에서** 처음부터 설명합니다.

## 1. 이 프로젝트를 한 문장으로

> **"NVIDIA GPU 한 장을 여러 사람(컨테이너)이 나눠 쓰도록, 각자에게 메모리 사용 한도(quota)를 정해주고 강제하는 시스템."**

조금 더 풀어볼게요.

### 문제 상황부터

여러분이 연구실이나 회사에서 값비싼 GPU(예: RTX 4060) 한 장을 **여러 명이 같이** 써야 한다고 상상해 보세요. GPU 메모리는 8GB뿐인데, A는 딥러닝 학습을, B는 추론을, C는 실험을 동시에 돌리고 싶어 합니다.

문제는 — GPU는 원래 **"먼저 잡는 사람이 다 가져가는"** 구조예요. A가 욕심내서 8GB를 몽땅 잡아버리면 B, C는 한 톨도 못 씁니다. 마치 공용 냉장고에 한 사람이 자기 음식으로 꽉 채워버리는 것과 같아요.

### 해결책: "분수(fractional) GPU"

그래서 이런 규칙을 만들고 싶어집니다:

> "A는 GPU의 40%(약 3.2GB)까지만, B는 60%(약 4.8GB)까지만 쓸 수 있어."

이렇게 **GPU를 분수(0.4, 0.6처럼)로 쪼개서** 각자에게 할당하는 것이 이 프로젝트의 이름 **fGPU(fractional GPU)** 의 뜻입니다.

이건 실제로 **Backend.AI**라는 상용 제품이 하는 일이고, 이 프로젝트는 그 핵심 메커니즘을 **직접 만들어보며 배우는 캡스톤/연구 프로토타입**입니다.

### 그런데 어떻게 강제하죠?

여기가 이 프로젝트의 **가장 창의적이고 핵심적인 아이디어**입니다.

GPU 메모리를 빌리려면 프로그램은 반드시 `cudaMalloc`이라는 **NVIDIA가 만든 함수**를 호출해야 합니다 (C의 `malloc`의 GPU 버전이에요 — 1장에서 배웁니다).

이 프로젝트는 그 `cudaMalloc` 호출을 **몰래 중간에서 가로챕니다.** 사용자가 "3GB 줘!"라고 요청하면, 우리 코드가 먼저 끼어들어서:

1. "지금 이 컨테이너가 이미 얼마나 썼지?" 확인하고
2. "이번 요청까지 더하면 한도를 넘나?" 계산하고
3. 넘으면 **거절(DENY)**, 안 넘으면 진짜 `cudaMalloc`에 넘겨서 **허용(ALLOW)**

이 "가로채기" 기술이 바로 **LD_PRELOAD**입니다 (4장의 주인공이에요). 놀랍게도 **사용자 프로그램은 코드를 한 줄도 안 고쳐도** 됩니다.

## 2. 전체 부품 지도

이 프로젝트는 크게 **세 덩어리**로 되어 있습니다. 지금은 이름만 눈에 익혀두세요.

```
┌──────────────────────────────────────────────────────────────┐
│  ① 훅 (The Hook)  ─ C 언어로 작성                              │
│     hook/src/fgpu_hook.c                                       │
│     GPU 메모리 함수를 가로채서 한도를 강제하는 "심장"          │
│     → 컴파일하면 libfgpu.so 라는 파일이 됩니다                 │
└──────────────────────────────────────────────────────────────┘
              ▲ 컨테이너 안에 주입됨 (LD_PRELOAD)
              │
┌──────────────────────────────────────────────────────────────┐
│  ② 백엔드 (The Backend) ─ Python FastAPI                       │
│     backend/                                                   │
│     "세션(컨테이너) 만들어줘/멈춰줘/목록 보여줘" REST API       │
│     Spring 으로 치면 @RestController + Service + Repository    │
│     Docker 를 조종해서 컨테이너를 띄우고, 훅을 주입합니다      │
└──────────────────────────────────────────────────────────────┘
              │ 조종
              ▼
┌──────────────────────────────────────────────────────────────┐
│  ③ 컨테이너 & 스크립트 (Containers & Scripts)                  │
│     runtime-image/, scripts/                                   │
│     GPU 프로그램이 실제로 돌아가는 격리된 상자(Docker)와       │
│     빌드·실행·검증을 자동화하는 .sh 스크립트들                 │
└──────────────────────────────────────────────────────────────┘
```

### 세 겹의 "문지기" (아주 중요한 큰 그림)

이 프로젝트를 진짜로 이해했다는 건 **"자원을 지키는 문지기가 세 명"** 이라는 걸 아는 것입니다. 이건 8장에서 자세히 다루지만 미리 그림을 심어둘게요:

| 문지기 | 언제 검사하나 | 무엇을 막나 | 어디에 있나 |
|--------|----------------|-------------|-------------|
| **① 어드미션(Admission)** | 컨테이너를 **만들기 직전** | "이미 꽉 찼는데 또 만들려고?" (지분 합이 1.0 초과) | 백엔드 (Python) |
| **② 훅(Hook)** | 프로그램이 **메모리를 요청할 때마다** | "네 컨테이너 한도 초과!" | libfgpu.so (C) |
| **③ 드라이버(Driver)** | 물리적 한계 | 진짜로 GPU 메모리가 바닥남 | NVIDIA 드라이버 (우리가 못 건드림) |

①②는 우리가 만든 문지기이고, ③은 원래 있던 물리 한계예요. 이 세 층이 협력해서 GPU를 안전하게 나눠 씁니다.

## 4. 미리 익히는 핵심 용어집

읽다가 막히면 여기로 돌아오세요.

| 용어 | 아주 쉬운 뜻 |
|------|--------------|
| **GPU** | 그래픽/AI 계산을 초고속으로 하는 부품. 여기선 이걸 나눠 쓰는 게 목표. |
| **CUDA** | NVIDIA GPU를 프로그램에서 다루기 위한 도구 모음(라이브러리). |
| **`cudaMalloc`** | "GPU 메모리 좀 빌려줘" 함수. C의 `malloc`의 GPU 버전. |
| **훅(Hook)** | 함수를 중간에서 몰래 가로채는 것. 이 프로젝트의 핵심 기법. |
| **LD_PRELOAD** | "내 라이브러리를 남들보다 먼저 로드해라"는 리눅스 환경변수. 가로채기를 가능하게 함. |
| **`.so` 파일** | 리눅스의 공유 라이브러리. Windows `.dll`의 사촌. `libfgpu.so`가 우리 훅. |
| **쿼터(quota)** | 사용 한도. "너는 3.2GB까지"처럼. |
| **비율(ratio)** | 쿼터를 전체의 몇 %로 줄지. `0.4` = 40%. |
| **컨테이너(Container)** | 격리된 실행 상자. 한 명의 사용자 = 한 컨테이너라고 생각하세요. |
| **Docker** | 컨테이너를 만들고 관리하는 도구. |
| **세션(Session)** | 이 프로젝트에서 컨테이너 하나 = 세션 하나. 백엔드가 관리하는 단위. |
| **어드미션(Admission)** | 새 세션을 받아줄지 말지 정하는 입구 심사. |
| **스테이지(Stage)** | 이 프로젝트의 개발 단계(1~12). 한 단계씩 검증하며 쌓았어요. |
| **FastAPI** | Python용 웹 프레임워크. Spring Boot의 Python 사촌. |

## 5. 이 프로젝트만의 문화 (알아두면 좋아요)

- **스테이지(단계)별 개발**: 기능을 Stage 1 → 2 → ... → 12 순서로 하나씩 완성하고, **각 단계를 검증한 뒤에만** 다음으로 넘어갑니다. 프로젝트 지침 파일인 [CLAUDE.md](../../CLAUDE.md)에 "다음(다음 단계)"이라고 말하기 전엔 앞서가지 말라는 규칙이 있어요.
- **환경변수로 설정 주입**: `FGPU_RATIO=0.4` 처럼 환경변수로 동작을 바꿉니다. 이게 훅·백엔드·스크립트를 관통하는 공통 관례예요 (2장에서 배웁니다).
- **로그는 `[fgpu]` 접두사**: 훅이 남기는 모든 로그는 `[fgpu]`로 시작합니다. 검증할 때 이걸 눈으로 찾거나 `grep`으로 걸러요.
- **한글 주석**: 핵심 파일([fgpu_hook.c](../../hook/src/fgpu_hook.c), [description.md](../../description.md))은 학습용으로 한글 주석이 아주 풍부합니다. 실제로 [fgpu_hook.c:1-63](../../hook/src/fgpu_hook.c#L1-L63)만 읽어도 프로젝트 요약이 됩니다.
