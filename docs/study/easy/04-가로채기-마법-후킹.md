# 4장. 가로채기 마법! (후킹) 🪄

> 이 장에서 배울 것: 주문서가 GPU로 가는 길목에서 우리가 먼저 가로채서 검사해요. 이게 이 책에서 가장 중요한 마법이에요!

## 🎯 오늘의 목표

1. "가로채기"가 무슨 말인지 알기
2. 컴퓨터가 어떻게 우리 도구를 먼저 읽는지 알기
3. 주문이 통과(ALLOW)되거나 거절(DENY)되는 게 어떤 상황인지 알기

---

## 택배 아저씨 이야기

상상해 봐요.

내가 온라인 쇼핑몰에서 뭔가를 주문했어요.
원래는 택배 아저씨가 창고에서 바로 물건을 가져다줘요.

그런데 어떤 날, **검사원 아주머니**가 택배 아저씨 앞에 서 계세요.
아주머니가 모든 택배 상자를 하나씩 열어봐요.
"이건 규격에 맞네, 통과!" 하면 물건이 배달돼요.
"이건 너무 크고 위험해, 안 돼!" 하면 배달이 안 돼요.

```
보내는 사람 -> [검사원 아주머니] -> 택배 아저씨 -> 받는 사람
                    ^
              이 분이 가로채는 분!
```

우리 프로그램도 이 검사원 아주머니처럼 일해요.
프로그램이 GPU에게 "메모리 줘!"라고 주문을 보낼 때,
**그 주문이 GPU로 가는 길에 우리가 먼저 가로채서 검사**해요.

---

## "먼저 읽기 목록" — 어떻게 가로채는 게 가능할까요?

자, 이제 진짜 비밀을 알려줄게요.

컴퓨터는 프로그램을 켤 때 이렇게 생각해요.

> "이 프로그램에 필요한 도구(부품 파일)들을 어떤 순서로 읽을까?"

그 순서가 적힌 **목록**이 있어요.
우리가 그 목록 **맨 앞**에 우리 파일을 써 놓으면,
진짜 GPU 도구보다 **우리 것이 먼저 끼어들어요.**

```
목록 맨 앞: [우리 도구]  <- 맨 먼저 읽힘!
목록 그 다음: [진짜 GPU 도구]
```

이렇게 되면 프로그램이 GPU에 주문할 때,
우리 도구가 제일 먼저 받아서 검사할 수 있어요.

이 "먼저 읽기 목록"을 컴퓨터 말로 **LD_PRELOAD**라고 해요.
이름은 외울 필요 없어요.
그냥 **"먼저 끼어들기 목록"** 이라고 기억하면 충분해요.

---

## 가짜 점원 비유로 한 번 더 봐요

식당 비유로 설명할게요.

원래 손님이 주문하면 **진짜 주방 아저씨**가 음식을 만들어요.
그런데 누군가가 **가짜 점원**을 주방 앞에 세워 놨어요.
손님이 주문하면 가짜 점원이 **먼저** 받아요.
가짜 점원이 검사를 하고 나서, 괜찮으면 진짜 주방 아저씨한테 넘겨줘요.

```
손님 (프로그램) -> [가짜 점원] -> 진짜 주방 아저씨 (GPU)
                      ^
             먼저 끼어드는 우리!
```

"먼저 끼어들기 목록"에 우리 이름을 써 둔 덕분에,
가짜 점원(우리 도구)이 항상 가장 먼저 서 있을 수 있어요.

---

## 우리 가로채기 도구의 이름은 `libfgpu.so`

우리가 만든 가로채기 도구는 **`libfgpu.so`**(리브에프지피유점에스오)라는 파일이에요.

`.so`는 **혼자서는 못 켜지고, 다른 프로그램 옆에 붙어서 도와주는 부품 파일**이에요.
레고 한 조각처럼 다른 프로그램에 끼워 쓰는 거예요.

파일 이름을 풀면 이래요.

- `lib` = 도구 모음 (자주 쓰는 도구들을 모아둔 상자)
- `fgpu` = fractional GPU (비율로 나눠 쓰는 GPU)
- `.so` = 다른 프로그램에 붙어서 도와주는 부품 파일

한 줄로 말하면: **"비율로 GPU를 나눠 쓰기 위한 가로채기 도구 파일"**이에요.

```
  [libfgpu.so 가 하는 일]

  +----------------------------------+
  |  1. 주문이 들어오면 먼저 받는다   |
  |  2. "양이 괜찮나?" 검사한다       |
  |  3. 괜찮으면 진짜 GPU에 넘긴다    |
  |  4. 너무 많으면 거절한다          |
  +----------------------------------+
```

---

## 가로채면 무슨 검사를 해요?

주문이 들어오면 `libfgpu.so`는 세 가지를 차례로 확인해요.

```
  1단계: 지금 이 친구가 이미 쓰고 있는 양은 얼마지?
  2단계: 이번에 달라는 양은 얼마지?
  3단계: 둘을 더했을 때 이 친구의 몫보다 많은가?
```

결론은 두 가지예요.

- **안 넘어**: "통과! (ALLOW)" — 진짜 GPU에 주문을 넘겨줘요.
- **넘어**: "거절! (DENY)" — 주문을 막아요. GPU는 아무것도 모르고 있어요.

```
주문이 들어옴
    |
    v
1단계: 지금 쓰는 양 확인    (예: 256 MiB)
    |
    v
2단계: 이번에 달라는 양 확인  (예: 6144 MiB)
    |
    v
3단계: 256 + 6144 = 6400 MiB, 내 몫(3276 MiB)보다 많아?
    |
  YES (많아!)          NO (괜찮아!)
    |                       |
    v                       v
  DENY                   ALLOW
(거절, 에러 반환)      (통과, GPU에 넘김)
```

## 실제 로그를 보면 이렇게 생겼어요

이 프로그램이 실제로 동작하면 화면에 이런 글자들이 나와요.
(컴퓨터 화면에 찍히는 메모를 "로그"라고 해요.)

```
[fgpu] ALLOW cudaMalloc size=256MiB  used=256/3276 MiB
[fgpu] DENY  cudaMalloc size=6144MiB used=256/3276 MiB
[fgpu] FREE  size=256MiB             used=0/3276 MiB
```

읽는 법.
- `ALLOW` = "통과!"
- `DENY` = "거절!"
- `FREE` = "자리 돌려줌"
- `used=256/3276` = "내 몫 3276 MiB 중에 256 MiB 쓰는 중"

여기서 **3276**이 어디서 왔는지 궁금할 거예요.
MiB(메비바이트)는 MB(메가바이트)랑 거의 같은데, 컴퓨터가 세기 편한 방식이에요.
1024 MiB가 약 1GB라서, 8GB는 약 8192 MiB가 돼요.
거기에 0.4를 곱하면 `8192 × 0.4 ≒ 3276 MiB`가 내 몫(quota)이에요.

> (사실은 더 복잡하지만 지금은 "전체 크기 × 비율 = 내 몫"으로 이해하면 충분해요.)

---

## 📦 한 줄 정리

`libfgpu.so`는 "먼저 끼어들기 목록" 덕분에 GPU 주문서를 **중간에 가로채서** 검사하고,
몫을 넘으면 DENY, 괜찮으면 ALLOW해요.

## ❓ 스스로 물어보기

**Q1. 우리 가로채기 도구 파일의 이름은?**
👉 `libfgpu.so`예요.

**Q2. ALLOW와 DENY는 각각 어떤 뜻이에요?**
👉 ALLOW는 "통과! GPU에 주문 넘겨줘", DENY는 "거절! 너무 많이 신청했어"예요.

**Q3. "먼저 끼어들기 목록"은 어떤 역할을 해요?**
👉 컴퓨터가 프로그램을 켤 때 우리 도구를 맨 먼저 읽게 해줘요. 그래서 가로채기가 가능해요.
