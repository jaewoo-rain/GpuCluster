# 6장. 각자의 방, 컨테이너 🏠

> 이 장에서 배울 것: 친구들이 GPU를 쓸 때 서로 방해 안 하도록 각자 칸막이 방을 줘요.

## 🎯 오늘의 목표

1. "컨테이너"가 뭔지 알기
2. 방마다 가로채기 도구가 따로 있다는 걸 알기
3. 방들이 서로 모른다는 게 왜 중요한지 알기

---

## 독서실 칸막이 이야기

공부방이나 도서관에서 **칸막이 책상**을 본 적 있나요?
한 칸 한 칸이 작은 방처럼 나뉘어 있어요.

내 칸에서 공부하면 옆 친구 소리가 덜 들리고,
내 물건이 옆 칸으로 넘어가지도 않아요.
각자 자기 칸에서 따로따로 공부할 수 있어요.

```
  독서실 칸막이

  +---------+   +---------+   +---------+
  |  내 칸  |   | 친구칸  |   | 친구칸  |
  | 공부중  |   | 공부중  |   | 공부중  |
  +---------+   +---------+   +---------+
    (서로 방해 안 함!)
```

컴퓨터에서 이런 "칸막이 방"을 **컨테이너(container)**라고 불러요.
컨테이너는 "여러 프로그램이 서로 방해 안 하도록 따로 나눈 공간"이에요.

---

## 방을 만들어 주는 도구 — Docker

"그럼 방은 누가 만들어요?"

방을 만들어 주는 도구 이름이 **Docker(도커)**예요.
"도커"는 영어로 항구에서 짐을 내리는 사람인데,
컴퓨터에서는 "프로그램이 살 수 있는 방을 척척 만들어 주는 도구"예요.

```
Docker (도커) = 방 만들어 주는 도구

  [방 주세요!] -> Docker -> [여기 새 방이에요!]
                              +--------------+
                              | 컨테이너 방  |
                              | 프로그램 살  |
                              | 수 있어요!   |
                              +--------------+
```

도커가 방을 만들면, 그 방 안에서 프로그램이 돌아요.
방 안의 프로그램은 다른 방에 무슨 프로그램이 있는지 몰라요.
각자 "내가 이 컴퓨터 전체를 쓰는 것 같다"고 느껴요.

---

## 방마다 가로채기 도구를 넣어요

이제 중요한 부분이에요!

방이 만들어질 때, 그 방의 **"먼저 끼어들기 목록"** 맨 앞에 `libfgpu.so`를 적어 넣어줘요.
(4장에서 배운 "먼저 끼어들기 목록" 기억하죠?)

그래서 그 방 안 프로그램의 주문은 자동으로 우리 도구를 먼저 거쳐요.
**각 방에는 자기만의 가로채기 도구**가 있는 셈이에요.

```
  컨테이너 A              컨테이너 B
  +------------------+   +------------------+
  |  프로그램 A      |   |  프로그램 B      |
  |        |         |   |        |         |
  |        v         |   |        v         |
  | [libfgpu.so A]   |   | [libfgpu.so B]   |
  | quota 0.4 (내 몫)|   | quota 0.6 (내 몫)|
  |        |         |   |        |         |
  +--------|----------+   +--------|----------+
           |                       |
           v                       v
           (서로 독립!)     (서로 독립!)
                   |       |
                   v       v
                 [ GPU ]
```

A의 가로채기 도구는 A의 몫(0.4)만 알아요.
B의 가로채기 도구는 B의 몫(0.6)만 알아요.
둘은 서로 무슨 주문을 하는지 전혀 몰라요!

---

## 왜 각자 따로 아는 게 좋아요?

서로 모르니까 **섞이지 않아요.**

예를 들어, 친구 A가 프로그램을 실수로 잘못 짜서 메모리를 많이 달라고 해도,
A의 가로채기 도구가 "이건 A 몫을 넘어!" 하고 막아줘요.
그 실수가 친구 B에게는 영향이 없어요.
B는 자기 몫(0.6)을 그대로 쓸 수 있어요.

```
  친구 A가 실수를 해도:

  컨테이너 A          컨테이너 B
  +------------+      +------------+
  | DENY!      |      | OK!        |
  | (A 몫 초과)|      | (영향 없음)|
  +------------+      +------------+
  A만 에러 받음        B는 멀쩡히 일함
```

이게 컨테이너의 힘이에요.
**각자 칸막이 방에 살아서 서로 영향을 안 줘요.**

---

## 📦 한 줄 정리

Docker가 만들어 주는 컨테이너(방)마다
`libfgpu.so`가 들어 있어서 각자 따로 quota를 지켜요.

**Q1. 컨테이너를 독서실에 비유하면?**
👉 독서실 칸막이예요. 각자 방이 따로 있어서 서로 방해 안 해요.

**Q2. 방을 만들어 주는 도구 이름은?**
👉 Docker(도커)예요.

**Q3. A의 `libfgpu.so`는 B가 얼마나 쓰는지 알까요?**
👉 몰라요. 각자의 도구는 자기 방의 몫만 알아요.
