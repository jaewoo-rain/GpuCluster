# 7장. 접수 창구, 서버 📋

> 이 장에서 배울 것: "GPU 방 주세요!" 신청을 받아주는 친절한 접수처가 있어요.

## 🎯 오늘의 목표

1. "백엔드 서버"가 뭔지 알기
2. 방을 신청하면 어떤 순서로 일이 이루어지는지 알기
3. 번호표(세션 id)가 왜 필요한지 알기

---

## 도서관 사서 선생님 이야기

학교 도서관에 가면 **사서 선생님**이 계시죠?

책을 빌리고 싶으면 사서 선생님께 말씀드려요.
"선생님, 이 책 빌려주세요."
선생님이 책을 찾아서 빌려주고, **날짜와 이름을 적어두세요.**
나중에 책을 반납하면 선생님이 기록을 지워줘요.

```
  나 → [사서 선생님] → 책장에서 책 꺼내줌
           ↑
     신청을 받고 기록하는 분
```

우리 프로그램에도 이런 **접수 창구** 역할을 하는 부분이 있어요.
이걸 **백엔드 서버**(Backend Server)라고 불러요.
어려운 이름이지만 뜻은 간단해요.
**"뒤에서(백) 일을 처리해 주는(엔드) 컴퓨터 프로그램"** 이에요.

우리 프로그램에서는 **FastAPI**(패스트에이피아이)라는 도구로 만들었어요.
FastAPI는 레고처럼 서버를 빠르게 만들 수 있게 해주는 도구예요.

## 신청서를 받아서 방을 만들어 줘요

누군가가 "GPU 방 주세요!"라고 신청하면,
백엔드 서버는 이렇게 일해요.

```
순서 1. 신청서를 받아요
  "ratio 0.4짜리 방 하나 주세요!"

순서 2. Docker(방 만들기 도구)에게 시켜요
  "Docker야, ratio 0.4짜리 방 만들어!"

순서 3. Docker가 방을 만들어요
  (컨테이너 방이 생기고, libfgpu.so도 넣어줌)

순서 4. 번호표를 줘요
  "방이 생겼어요! 번호는 'abc-123'이에요."
```

그림으로 보면:

```
  신청하는 사람
       │
       ▼
  [백엔드 서버] ← 접수 창구
       │
       ▼
  [Docker] ← 방 만들기 도구
       │
       ▼
  [컨테이너 방] ← 새로 생긴 방
  (libfgpu.so + quota 들어있음)
```

## 번호표 — 세션 id

방을 만들면 서버가 **번호표**를 줘요.
이 번호표를 **세션 id**(세션 아이디)라고 해요.

왜 번호표가 필요할까요?

식당에서 웨이팅 번호표를 받아본 적 있나요?
번호표가 있어야 "제 테이블 어디예요?"라고 물어볼 수 있잖아요.
GPU 방도 마찬가지예요.
번호표(세션 id)가 있어야 나중에 "내 방 어떻게 됐어?"라고 확인하거나,
"내 방 없애줘!"라고 말할 수 있어요.

```
  번호표 예시:
  세션 id = "a3f7bc29-1234-..."

  이 번호로 할 수 있는 것:
  - "이 방 지금 어떤 상태야?" (확인)
  - "이 방에서 뭐가 나왔어?" (로그 보기)
  - "이 방 멈춰줘!" (중지)
  - "이 방 없애줘!" (삭제)
```

## 웹 화면에서도 볼 수 있어요

서버가 잘 동작하면 **웹 화면**에서도 방을 신청하고 볼 수 있어요.
인터넷 주소창에 `http://localhost:8000/`을 치면
방 목록이 쫙 나오고, 새 방을 신청하는 버튼도 있어요.

```
  웹 화면 (브라우저)
  ┌──────────────────────────────────┐
  │  GPU 방 목록                      │
  │  ┌────────────────────────────┐   │
  │  │ 방번호: a3f7... | 상태: 실행중 │ │
  │  └────────────────────────────┘   │
  │  [새 방 신청하기 버튼]             │
  └──────────────────────────────────┘
```

## 방이 너무 많으면 미리 막아줘요

서버는 똑똑해서, **비율 합이 1.0을 넘으면 미리 거절**해요.
(5장에서 배운 것 기억나죠? 0.4 + 0.6 = 1.0이면 딱 맞는 것)

이미 0.7짜리 방이 있는데 0.6짜리를 또 달라고 하면:
0.7 + 0.6 = 1.3 → 1.0을 넘어!
이럴 때 서버가 "미안, 지금은 공간이 없어요!"라고 거절해요.

이 기능을 **입장 조절**이라고 해요.
5장에서 잠깐 나온 그 이야기예요!
학교 행사에서 "정원이 꽉 찼어요, 못 들어와요!"와 같아요.

## 📦 한 줄 정리

백엔드 서버는 "GPU 방 주세요!" 신청을 받아서 Docker에게 방을 만들게 하고,
번호표(세션 id)를 나눠줘요.

---

## ❓ 스스로 물어보기

**Q1. 백엔드 서버를 도서관에 비유하면 누구예요?**
👉 사서 선생님이에요. 신청을 받고 책(방)을 찾아줘요.

**Q2. 세션 id는 왜 필요해요?**
👉 내 방이 어떤 방인지 나중에 확인하거나 없애달라고 할 때 쓰려고요. 번호표예요.

**Q3. 비율 합이 1.0을 넘으면 어떻게 돼요?**
👉 서버가 미리 거절해요. "공간이 없어요!"
