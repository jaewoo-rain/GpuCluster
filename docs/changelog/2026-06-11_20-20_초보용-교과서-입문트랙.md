# 초보용 교과서 입문 트랙 추가 (docs/study/easy/)

날짜: 2026-06-11

## 무엇을

코딩·C·컴퓨터 구조 지식이 0인 초등학생을 독자로 하는 GpuCluster 입문 교과서 트랙을 `docs/study/easy/` 에 한국어로 완성했다. 기존 `docs/study/*.md`(어른용)와 별개의 "아주 쉬운 버전"이다.

## 작업 방식

`easy-textbook-writer` 가 집필하고 `beginner-comprehension-reviewer` 가 초등학생 입장에서 검수한 뒤, 막히는 부분을 다시 집필자가 고치는 **집필→검수→수정 루프**로 진행했다.

- 1차 검수: 04·06·08·09·10장 반려 (LD_PRELOAD 원리 블랙박스, ASCII 정렬 깨짐, 흐름도 미로, SM 원어 노출 등)
- 2차 검수: 02·04·09장 반려 (소수 중의성, MiB 단위 미설명, 흐름도 6 vs 요약 10 단계 불일치)
- 3차 검수: 세 챕터 모두 통과 → 전 챕터 통과로 루프 종료

사실 정확성은 `hook/src/fgpu_hook.c`, `CLAUDE.md`, `docs/study/15-limitations.md`, `docs/study/12-launch-monitoring.md` 로 확인했다(libfgpu.so 이름, ALLOW/DENY 로그, 가로채는 함수들, quota=ratio×전체메모리, throttle 100ms 윈도우/nanosleep/커널 비-드롭, admission 비율 합 ≤ 1.0). 쉽게 풀되 거짓 없이.

## 각 챕터 핵심 비유

| 챕터 | 핵심 비유 |
|---|---|
| 01 컴퓨터와 두뇌 GPU | 똑똑한 한 명(CPU) vs 평범하지만 1000명(GPU) — 1명이 1000문제 vs 1000명이 1문제씩 |
| 02 왜 나눠쓰나 | 큰 케이크/운동장 나눠 쓰기 — fractional = 점이 찍힌 숫자(0.4/0.6)로 쪼개기 |
| 03 GPU에게 일시키기 | 식당 주문서 — `cudaMalloc`="메모리 줘", `cudaFree`="돌려줄게" |
| 04 가로채기 마법(후킹) | 택배를 길목에서 가로채 검사 — '먼저 끼어들기 목록'(LD_PRELOAD)에 libfgpu.so를 맨 앞에 |
| 05 메모리 칸 나눠주기 | 사물함 칸 / 급식 정량 — 내 몫(quota) 넘으면 거절 |
| 06 각자 방 컨테이너 | 칸막이 독서실 — Docker가 방을 만들고 방마다 libfgpu.so를 넣어줌 |
| 07 접수창구 서버 | 도서관 사서 — 신청 받기 → 방 만들기 → 번호표(세션 id) |
| 08 전체 설계도 | 큰 세로 흐름도 — 사람→웹화면→서버→Docker→컨테이너(libfgpu.so+GPU), 두 겹 방어막 |
| 09 신청이 흘러가는 길 | 편지 배달 여행 — 버튼 클릭부터 결과까지 6단계 단방향 흐름도 |
| 10 같이쓰면 빨라질까 | 놀이터 그네 순서 — 메모리는 나뉘지만 일꾼은 공유, throttle로 100ms 안에서 비율만큼 번갈아 |
| 11 아직 못 하는 것 | 솔직한 고백 — 계산 일꾼 칸막이·뒷문 프로그램 차단·MIG급 격리는 못 하지만 일반 GPU 나눠쓰기엔 충분 |

## 강조 사항 반영

08·09장은 "서비스 구성·상호작용·설계도"가 핵심이라, 검수에서 ASCII 그림을 가로·세로 혼합 → **위→아래 단방향 흐름**으로 재설계하고 단계 수를 일치시켰다.

## 구체성 보강 (추가 라운드)

"비유만 많고 실물이 없다(추상적)"는 피드백을 받아 10장을 실측 데이터로 보강했다.

- 10장에 `experiments/sharing_results/`의 **실제 측정값**(작은 챗봇 AI Qwen2-0.5B 기준)을 넣었다: 혼자 쓸 때 약 27 tokens/s → A와 같이 쓸 때 약 18 tokens/s(A는 약 15). "둘이 같이 쓰면 느려진다"를 말이 아닌 숫자로 증명.
- 막연한 "AI 프로그램" → 01·03장에 "사진 속 고양이 알아맞히기 / 글 써주는 챗봇" 같은 구체 예시 한 줄씩 추가.
- 검수에서 잡힌 사실 오류 2건 수정: ① "토큰은 낱글자보다 작은 조각"(틀림) → "글자나 짧은 낱말 조각". ② throttle 막대 그림이 A/B가 짠 듯이 엇갈리는 것처럼 보이던 것 → "각 방에 따로 거는 것, 둘 다 동시에 쉬면 GPU가 잠깐 논다(work-conserving 아님)"로 정정(11장 한계와 일관).
- 비율(0.4/0.6)은 메모리를 나누는 약속일 뿐 계산 일꾼을 나누는 게 아니라는 점을 한 줄로 명시(왜 B가 그렇게 느려지나 의문 해소).
