# 10장. 같이 쓰면 빨라질까? 우리 실험 🔬

> 이 장에서 배울 것: 두 명이 GPU를 같이 쓰면 어떻게 되는지 실제로 실험해 봤어요. 결과가 솔직하고 재미있어요!

## 🎯 오늘의 목표

1. GPU를 같이 쓸 때 어떤 일이 생기는지 알기
2. 메모리 칸은 잘 나뉘지만 "일꾼"은 함께 쓴다는 걸 알기
3. 번갈아 쓰기(throttle)가 무엇인지 알기

---

## 실험 이야기

우리가 만든 프로그램으로 실제로 실험했어요.

실험에는 **작은 챗봇 AI**를 썼어요.
이 챗봇은 질문을 받으면 글자 조각(토큰)을 한 개씩 차례로 만들어서 대답을 써요.
(토큰은 글자나 짧은 낱말 조각 정도예요. "안녕하세요"가 몇 개의 토큰으로 나뉜다고 생각하면 돼요.)

이 챗봇이 **1초에 몇 개의 글자 조각을 만드는지**가 빠르기예요.
많이 만들수록 대답이 빨리 나와요.

우리는 이 챗봇을 이렇게 두 가지 상황에서 실험했어요.

친구 A와 B가 같은 GPU를 나눠 써요.
- A는 비율 0.4 (전체의 40%)
- B는 비율 0.6 (전체의 60%)

결과가 어떻게 됐을까요?

---

## 메모리(책상 자리)는 잘 나뉘어요

먼저 좋은 소식이에요.

**메모리 나누기는 잘 됐어요.**

A가 자기 몫(40%)을 넘게 달라고 하면 -> DENY!
B가 자기 몫(60%) 안에서 달라고 하면 -> ALLOW!

둘이 같은 GPU를 쓰지만, 각자 자기 몫 안에서만 써요.
서로 방해하지 않고, 서로 칸을 빼앗지 않아요.

```
  GPU 메모리 (8GB)
  +----------------------------------+
  |   A 칸 (3.2GB) | B 칸 (4.8GB)   |
  |  A만 사용 가능 | B만 사용 가능  |
  +----------------------------------+
  "책상 자리는 깔끔하게 나뉨!"
```

이것만으로도 아주 유용해요.
A의 실수가 B에게 영향을 안 주고, B의 실수가 A에게 영향을 안 줘요.

---

## 그런데 계산 일꾼은 함께 써요

이제 솔직한 이야기예요.

GPU 안에는 **계산을 하는 일꾼들**이 있어요.
(우리는 그냥 "일꾼들"이라고 부를게요.)

이 일꾼들은 **칸막이로 나눌 수 없어요.**
A의 일과 B의 일이 같은 일꾼들을 두고 경쟁해요.

```
  A의 일 --+
            +--> [GPU 일꾼들] <-- 둘이 같이 씀!
  B의 일 --+
```

그래서 둘이 동시에 열심히 일하면 둘 다 느려져요.
진짜 측정한 숫자로 보여 드릴게요.

### 진짜 측정 결과 (1초에 글자조각 몇 개?)

(A는 비율 0.4, B는 비율 0.6으로 맞추고, 한 번 대답할 때 토큰 128개를 만들게 해서 쟀어요.)

```
상황                      | 1초에 글자조각 | 글자조각 128개 걸린 시간
--------------------------|--------------|----------------------
B가 혼자 GPU를 쓸 때      |  약 27개      |  약 4.7초
B가 A와 같이 쓸 때        |  약 18개      |  약 7.6초
A가 B와 같이 쓸 때        |  약 15개      |  약 11초
```

눈으로 한눈에 봐요.

```
  혼자 쓸 때 (B):  ███████████████████████████  약 27개/초
  같이 쓸 때 (B):  ██████████████████           약 18개/초
  같이 쓸 때 (A):  ███████████████              약 15개/초

  ◀ 적음                                  많음 ▶
```

B는 혼자 쓸 때 1초에 27조각을 척척 만들었어요.
그런데 A와 같이 쓰니까 18조각으로 떨어졌어요.
A는 15조각밖에 못 만들었어요.

여기서 중요한 점! 비율 0.4 / 0.6은 **책상(메모리)을 나누는 약속**이지, **계산 일꾼을 나누는 게 아니에요.** 일꾼은 A와 B가 똑같이 두고 경쟁해요. 그래서 A가 40%만 받았어도, 둘 다 열심히 일하면 일꾼을 나눠 갖느라 B도 많이 느려지는 거예요.

9장까지 말로만 했던 걸 이제 진짜 숫자로 확인했어요!

운동장 비유로 생각해요.
체육 시간에 1반이 축구를 하는데 2반도 같은 운동장에서 달리기를 해요.
운동장을 나눠 쓰긴 하지만, 좁아지면 서로 느려질 수 있잖아요.

---

## 한 명이 쉬엄쉬엄 쓰면 다른 친구가 빨라져요

재미있는 실험 결과도 있어요.

A가 쉬엄쉬엄 하고 있으면 (일을 조금씩 할 때),
일꾼들이 B의 일을 더 많이 도울 수 있어요.
그럼 B는 더 빨리 끝나요.

```
  A가 쉬는 중...

  일꾼들: "A 일이 없으니까 B 일 더 빨리 하자!"
  -> B가 빨라짐!
```

이건 비유하면, 쉬는 시간에 운동장이 텅 비면
나 혼자 뛸 때 훨씬 넓게 쓸 수 있잖아요.

---

## 번갈아 쓰기 — Throttle (쓰로틀)

그렇다면 "A가 너무 열심히 할 때 조금 쉬게 하면 어떨까?" 하는 아이디어가 생겨요.

이걸 **throttle**(쓰로틀)이라고 해요.
쉬운 말로 하면 **"번갈아 쓰기"** 또는 **"속도 조절"**이에요.

예를 들어 A에게 비율 0.4(40%)를 정해 주면,
A는 정한 시간의 40%만 일하고 나머지는 쉬어요.
혼자라면 1초에 약 27개를 만들 텐데, 0.4로 속도를 조절하면 약 10~11개 정도로 줄어드는 셈이에요.
"정한 비율만큼 느려진다"는 걸 숫자로 확인할 수 있어요.

---

## 그네 순서 정하기 비유

놀이터 그네가 하나밖에 없어요.
A와 B가 둘 다 그네를 타고 싶어요.

**공평하게 하는 방법:**
A가 1분 타면 -> 잠깐 기다려.
B가 1분 타면 -> 잠깐 기다려.
번갈아 타요!

쓰로틀이 **없을 때**는 B가 계속 쉬지 않고 그네를 탈 수 있어요.

```
시간 ->
B:  ████████████████████████  (계속 탐, 쉬지 않음)
```

쓰로틀이 **있을 때**는 A와 B 각자 정한 비율만큼만 일하고 쉬어요.
쓰로틀은 각 방에 따로 거는 거예요.
A는 40%만 일하고 쉬고, B는 60%만 일하고 쉬어요. 각자 정한 만큼만 일하는 거예요.
딱 맞춰 번갈아 도는 건 아니에요. 둘 다 동시에 쉴 때도 있어서 그땐 GPU가 잠깐 놀기도 해요.

```
시간 ->
A:  ████░░░░░░████░░░░░░  (40%만 일하고 나머지는 쉼)
B:  ██████░░░░██████░░░░  (60%만 일하고 나머지는 쉼)
    (두 줄이 따로 움직여요 — 칼같이 엇갈리진 않아요)
```

---

## 우리 프로그램의 번갈아 쓰기는 이렇게 해요

우리 가로채기 도구(`libfgpu.so`)는 GPU에 일을 시킬 때도 살짝 가로채요.
(GPU에 "계산해!"라고 시키는 것도 `cudaLaunchKernel`이라는 주문이거든요.)

이 주문을 가로채서, **정해진 비율만큼 시간이 지나면 잠깐 기다리게** 해요.

1초를 1000조각으로 나눈 것 중 하나가 **1ms(밀리초)**예요.
100ms면 1초의 10분의 1, 아주 짧은 시간이에요.

우리는 100ms 창 안에서 정해진 비율만큼만 허용해요.

```
  비율 0.4 일 때:
  100ms 창에서 40ms만 일하고, 나머지 60ms는 잠깐 기다림.

  +------------------------------------------+
  |            100ms 한 칸                    |
  | [일함: 40ms]     [기다림: 60ms]           |
  +------------------------------------------+
  -> 다음 100ms 창에서 또 반복
```

중요한 점: **주문 자체를 버리지는 않아요.**
잠깐 기다렸다가 반드시 GPU에 전달해요.
기다리는 것뿐, 없애는 건 없어요!

---

## 실험 결과 요약

| 상황 | 결과 |
|---|---|
| B 혼자 쓸 때 | 약 27 글자조각/초 (빠름) |
| B가 A와 같이 쓸 때 | 약 18 글자조각/초 (느려짐) |
| A와 B 둘 다 열심히 | 둘 다 느려져요 (계산 일꾼을 나눠 써서) |
| A가 쉬엄쉬엄 | B가 더 빨라져요 |
| 번갈아 쓰기(throttle) 켬 | A의 속도가 정한 비율만큼 줄어들어요 |
| 메모리 나누기 | 잘 됐어요! 서로 방해 없어요 |

---

## 📦 한 줄 정리

메모리 책상 자리는 잘 나뉘지만,
계산 일꾼은 함께 써요.
번갈아 쓰기(throttle)로 비율만큼 쉬게 하면
더 공평하게 나눌 수 있어요.

## ❓ 스스로 물어보기

**Q1. GPU 메모리 나누기는 잘 됐나요?**
👉 네! A 몫과 B 몫이 따로따로 지켜져요.

**Q2. A가 열심히 일할 때 B가 느려질 수 있는 이유는?**
👉 계산 일꾼들을 A와 B가 같이 쓰기 때문이에요. 칸막이가 없어요.

**Q3. throttle(번갈아 쓰기)이 주문을 없애나요?**
👉 아니요! 잠깐 기다리게 할 뿐, 주문 자체는 반드시 실행돼요.
