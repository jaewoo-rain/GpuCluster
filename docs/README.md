## 🚀 문서 사이트 실행법 (MkDocs + Material)

### 1) 설치 (최초 1회만)

```bash
pip install mkdocs-material
```

### 2) 로컬 미리보기 (자동 새로고침)

**반드시 저장소 루트**(= `mkdocs.yml` 이 있는 곳)에서 실행합니다.

```bash
cd C:\Users\양재우\Desktop\backend_ai\GpuCluster
python -m mkdocs serve -a 127.0.0.1:8080
```

→ 브라우저에서 **http://127.0.0.1:8080/** 접속.

- 📝 `.md` 파일이나 `mkdocs.yml` 을 **저장하면 자동으로 재빌드 + 브라우저 새로고침**됩니다 (live reload).
- ⏹ **끄기**: 실행한 터미널에서 `Ctrl + C`.
- 🔌 **포트 변경**: `python -m mkdocs serve -a 127.0.0.1:8081`
  (포트 `8000` 은 Windows 에서 접근이 막혀 있어 `8080` 을 씁니다.)

### 3) 정적 사이트로 빌드 (배포/공유용)

```bash
python -m mkdocs build
```

→ 완성된 HTML 이 `site/` 폴더에 생성됩니다. (`site/` 는 `.gitignore` 처리되어 커밋되지 않습니다.)

## 🔗 코드 링크 규칙 (중요)

교과서는 실제 소스 코드를 `../../hook/src/fgpu_hook.c#L128` 같은 **상대 경로 링크**로 가리킵니다.

- **VS Code / GitHub 저장소 뷰**에서는 상대 링크 그대로 동작합니다.
- **MkDocs 사이트**에서는 빌드 훅(`../mkdocs_hooks.py`)이 이 링크를 자동으로 **GitHub 소스 주소로 변환**하고 **새 탭**으로 열리게 합니다.

그래서 새 코드 링크를 추가할 때는 규칙이 있습니다:

> ✅ 코드 링크는 반드시 **`../../` 로 시작하는 상대 경로**로 쓰세요.
> 예: `[fgpu_hook.c:128](../../hook/src/fgpu_hook.c#L128)`
> (`../../` 없이 `backend/...` 처럼 쓰면 사이트에서 404 가 납니다.)

> ⚠️ GitHub 링크가 **정확한 줄**로 가려면 코드가 GitHub `main` 에 **push** 되어 있어야 합니다. 코드가 바뀌면 줄 번호가 밀리니, 코드 수정 후에는 문서의 줄 번호도 함께 확인하세요.
