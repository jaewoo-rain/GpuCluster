## 1. 무엇을 고쳤나

`SessionManager._create_locked` 가 컨테이너를 spawn 한 뒤 `store.insert` 사이에서 예외가 나면
이미 떠 있는 컨테이너(`remove=False`)가 고아로 남아 GPU/메모리/포트를 누수하고 admission 회계가
어긋나던 문제를 **(A) 보상정리(rollback)** 와 **(B) docker→DB orphan 스윕** 두 단계로 해결.

## 2. 변경 파일 및 내용

### `backend/app/services/session_manager.py` (담당: backend-api-engineer)

**Step A — 보상정리**
- import 추가: `sys`, `datetime.timedelta`, `typing.Set`
- 모듈 상수 `_ORPHAN_GRACE_SECONDS = 30` (line 64)
- `__init__`: 인스턴스 변수 `self._inflight_cids: Set[str] = set()` 추가 (line 88) — spawn 진행 중(insert 완료 전) 컨테이너 id 추적, 스윕 race 방지
- 신규 `_rollback_spawn(self, container_id, ws_dir)` (line 91–126):
  - `asyncio.shield(asyncio.to_thread(self.docker.remove_container, container_id, True))` — 취소 컨텍스트에서도 정리 완주
  - `docker.errors.NotFound` → pass, 그 외 예외 → swallow + stderr 경고(`[fgpu] rollback: ...`), **원래 예외 미마스킹**
  - `ws_dir` 주어지면 `shutil.rmtree(ws_dir, ignore_errors=True)`
- `_create_locked` (line 176–285):
  - `created_ws` 플래그 추가 — workspace mkdir 을 `exist_ok=False` 로 바꾸고 `FileExistsError` 시 `created_ws=False`(기존 노트북 보호)
  - `c` 반환 직후 `_inflight_cids.add(c.id)`, `try/finally` 로 `discard` 보장
  - `c` 반환 직후 ~ `store.insert` 성공까지 `try/except BaseException` → 예외 시 `_rollback_spawn(c.id, workspace_dir if created_ws else None)` 후 `raise`

**Step B — orphan 스윕**
- 신규 `reconcile_orphans(self, grace_seconds=_ORPHAN_GRACE_SECONDS) -> int` (line 364–479):
  - `list_fgpu_containers()` enumerate → `_inflight_cids` 제외 / `fgpu-` prefix 검증 / `store.get(sid)` 있으면 skip / `started_at` grace 이내거나 파싱 실패면 보수적 skip → 나머지 orphan 만 `remove_container(force=True)` 회수
  - daemon 예외 시 경고 로그 + 0 반환, `list_all` 등 정상 경로 미파손

### `backend/app/services/docker_manager.py` (담당: docker-runtime-engineer)
- `remove_container` docstring 보강 (line 192–201) — id 부재 시 `docker.errors.NotFound` raise, 호출측 swallow 계약 명시
- 신규 `list_fgpu_containers() -> list[dict]` (line 204–248):
  - `client.containers.list(all=True, filters={"name": "fgpu-"})`
  - 항목 형식 `{"id", "name"(앞 "/" strip), "status", "started_at"(State.StartedAt ISO8601 or None)}`
  - 개별 파싱 실패는 skip, daemon 예외는 전파(호출측 catch)
  - label 부여는 미적용(name prefix 1차 계약 유지, 후속 개선 여지로 남김)

### `backend/app/main.py` (담당: backend-api-engineer)
- startup 이벤트 `@app.on_event("startup")` 에서 `reconcile_orphans()` 1회 호출 (line 75–94) — `BaseException` 으로 감싸 실패해도 부팅 차단 안 함
